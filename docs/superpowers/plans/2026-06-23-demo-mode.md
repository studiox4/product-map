# Demo Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a no-auth, nothing-saves `/demo` that runs the real Hono backend in-page against in-memory PGlite, so anyone can try Product Map — create, edit, delete — with a fresh curated dataset every visit.

**Architecture:** The web app already talks to the API only through the Hono `hc` typed client in `apps/web/src/lib/api.ts`. In demo mode we swap that client's `fetch` to call the real Hono route logic (`demoApp.fetch`) bound to an in-memory PGlite (WASM Postgres) database. No Service Worker, no logic re-implementation, no production runtime branches — the demo bundle is selected via Vite aliases. In-memory DB ⇒ refresh resets ⇒ nothing saves.

**Tech Stack:** React 18 + Vite, React Router 6, TanStack Query, Hono `^4.6.14` (`app.fetch` + `hc({fetch})`), Drizzle ORM, `@electric-sql/pglite` + `drizzle-orm/pglite`.

## Strong Goals (definition of done — every task serves these)

1. **G1 — Zero persistence.** No demo action survives a page refresh. No real DB, network, cookie, or IndexedDB write. Verified by automated reset test.
2. **G2 — Full core fidelity.** Every core flow (features, roadmap horizons, board, ideas, docs edit, objectives, plans, releases, comments, votes, evidence, decisions, overview, activity) works through the *real* route logic — mutations included. No hand-written CRUD mocks.
3. **G3 — Boots fast and isolated.** `/demo` cold-loads the WASM bundle only on entry (never on marketing/landing/normal app); first interactive < 3s on a warm cache. Production bundle unaffected in size and behavior.
4. **G4 — Production untouched.** Prod build keeps node-postgres + real auth + CSRF. All demo behavior is alias-selected into the demo bundle only. CI proves it.
5. **G5 — Drift-proof.** A CI guard fails the build if a node-only module-level import reaches the demo path, or if a new route bypasses the demo db/auth assembly. The demo can't silently rot.
6. **G6 — Honest UX.** A persistent banner states nothing saves and links to sign-up; stubbed features (login, AI, export, uploads-persistence) degrade gracefully, never error in the user's face.

## Global Constraints

- Hono `^4.6.14`; do not bump. Rely only on `app.fetch(Request)` and `hc(url, { fetch })` — both stable in this version.
- Schema is the single source of truth in `packages/db/src/schema.ts`; do NOT fork it for demo. PGlite must run the same migrations/DDL.
- Demo-reachable code (anything imported by `apps/web/src/demo/**` or the demo Vite alias set) must have **no node-only module-level imports** (`node:*`, `pg`, `@node-rs/argon2`, `@aws-sdk/*`, `@ai-sdk/*`, `archiver`, `nodemailer`, `@hono/node-server`). Node usage allowed only behind dynamic `import()` inside a handler that the demo never calls.
- Demo auth identity is fixed: `{ id: 'demo-user', role: 'admin' }`. A `users` row with `id='demo-user'` and a `memberships` row for the demo project MUST be seeded.
- No new runtime `if (demoMode)` branches inside production route handlers. Selection happens at build time via Vite alias, or in `apps/web` demo modules only.

- [ ] **Step 1: Write the spike test**

```ts
import { describe, it, expect } from 'vitest';
import { Hono } from 'hono';
import { hc } from 'hono/client';

const app = new Hono()
  .get('/api/ping', (c) => c.json({ ok: true }))
  .post('/api/echo/:id', async (c) => {
    const body = await c.req.json();
    return c.json({ id: c.req.param('id'), body });
  });

const client = hc<typeof app>('http://demo.local', {
  fetch: (input: RequestInfo | URL, init?: RequestInit) =>
    app.fetch(new Request(input as string, init)),
});

describe('hono in-page fetch swap', () => {
  it('GET round-trips', async () => {
    const res = await client.api.ping.$get();
    expect(await res.json()).toEqual({ ok: true });
  });
  it('POST with param + body round-trips', async () => {
    const res = await client.api.echo[':id'].$post({ param: { id: 'x1' }, json: { a: 1 } });
    expect(await res.json()).toEqual({ id: 'x1', body: { a: 1 } });
  });
});
```
