# E1 Archive (soft-delete) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace hard-delete of projects and features with a restorable soft-delete (archive); permanent purge only from a trash view, only when already archived; archived items hidden from active lists and read-only.

**Architecture:** Add nullable `archivedAt` to `projects` + `features` (migration 0017). Add archive/restore routes; repurpose the `DELETE` endpoints to permanent purge gated on `archivedAt IS NOT NULL`. Hide archived rows from primary display lists with `archivedAt IS NULL`; expose archived sets via a `?archived=1` param. Enforce read-only on an archived project with a single guard in the project-scoped content mount. Web: relabel Delete→Archive and add archived/trash views with Restore + Delete-permanently.

**Tech Stack:** Drizzle ORM + Postgres (schema-first; migrations via `drizzle-kit`), Hono API, Zod, React + TanStack Query + Tailwind + shadcn/ui, Vitest (+ jsdom/testing-library for web).

## Global Constraints

- pnpm monorepo: `packages/db` (schema+migrations), `packages/shared`, `apps/api` (Hono), `apps/web` (React). Next migration = **0017**.
- Migrations are GENERATED: edit `packages/db/src/schema.ts`, then `pnpm --filter @productmap/db generate`. Never hand-write SQL.
- API/web tests need Postgres; the sandbox blocks DB/network — run test commands with the sandbox disabled. Postgres is running.
- `archivedAt`: nullable `timestamp('archived_at', { withTimezone: true })` on both tables (mirror `templates.archivedAt`, `schema.ts:225`). NULL = active.
- Permissions: project archive/restore/purge = **owner**; feature archive/restore = **editor**; feature purge = **owner**.
- Purge (`DELETE`) is allowed ONLY when the row is already archived; otherwise **409** with `{ error: 'not_archived' }`.
- Content write to an archived project → **409** `{ error: 'project_archived' }`.
- No raw in-app route string literals in web (CI grep gate); use `appRoutes` from `@/lib/routes`.
- TDD: failing test → watch fail → implement → watch pass → commit. Frequent commits.

Spec: `docs/superpowers/specs/2026-06-24-e1-archive-design.md`.

```ts
export function useArchivedFeatures() {
  const pid = useProjectId();
  return useQuery({
    queryKey: ['p', pid, 'features', 'archived'] as const,
    queryFn: () => fetchJson<FeatureWithDocs[]>(`${apiPath(pid, 'features')}?archived=1`),
    enabled: !!pid,
  });
}
export function useArchiveFeature() {
  const pid = useProjectId(); const qc = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => fetchJson<void>(apiPath(pid, 'features', id, 'archive'), { method: 'POST' }),
    onSettled: () => { qc.invalidateQueries({ queryKey: queryKeys.features(pid) }); qc.invalidateQueries({ queryKey: ['p', pid, 'features', 'archived'] }); },
  });
}
export function useRestoreFeature() {
  const pid = useProjectId(); const qc = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => fetchJson<void>(apiPath(pid, 'features', id, 'restore'), { method: 'POST' }),
    onSettled: () => { qc.invalidateQueries({ queryKey: queryKeys.features(pid) }); qc.invalidateQueries({ queryKey: ['p', pid, 'features', 'archived'] }); },
  });
}
export function usePurgeFeature() {
  const pid = useProjectId(); const qc = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => fetchJson<void>(apiPath(pid, 'features', id), { method: 'DELETE' }),
    onSettled: () => qc.invalidateQueries({ queryKey: ['p', pid, 'features', 'archived'] }),
  });
}
```
